# Установка Bitrix24: один сценарий через страницу мастера

Нужная страница:

```text
https://<SITE_DOMAIN>/panel/bitrix
```

На ней уже есть автоподстановка текущих значений из сервера, готовый `.env`, готовые URL и короткая инструкция.

## 1. Сгенерируй все данные на странице

1. Открой `https://<SITE_DOMAIN>/panel/bitrix`.
2. В блоке `1. Генерация Данных и Путей` укажи:
- `SITE_DOMAIN`
- `ACME_EMAIL`
3. Если токены пустые, нажми `Сгенерировать Отсутствующие Токены`.
4. Нажми `Копировать .env Блок`.

## 2. Примени `.env` на сервере

Вставь скопированный блок в `.env`, затем перезапусти:

```bash
docker compose up --build -d
```

Важно:
- DNS `A` запись `SITE_DOMAIN` должна указывать на IP сервера.
- Порты `80` и `443` должны быть открыты.

## 3. Заполни Bitrix без ручного поиска путей

На этой же странице в блоке `Что заполнить в Bitrix локальном приложении` уже готово:
- `Путь вашего обработчика`
- `Путь для первоначальной установки`
- `Webhook Open Lines`
- право приложения: `imconnector`

Скопируй и вставь эти данные в Bitrix.

## 4. Подключи портал одной кнопкой

После установки Bitrix откроет эту же страницу с `AUTH_ID/REFRESH_ID/APP_SID` и токены подставятся в поля.

Осталось:
1. Выбрать `Менеджер`.
2. Указать `line_id`.
3. Нажать `Подключить`.

## 5. Проверка

1. Отправь сообщение из Telegram клиенту.
2. Убедись, что оно появилось в Open Lines.
3. Ответь оператором в Open Lines.
4. Убедись, что ответ пришел в Telegram.

## 6. Важно про HTTPS

- Не используй `https://<домен>:8090`.
- Используй:
  - `https://<SITE_DOMAIN>/panel/bitrix`
  - `https://<SITE_DOMAIN>/telegram/`
