# Установка Bitrix24: один сценарий через страницу мастера

Нужная страница:

```text
https://<SITE_DOMAIN>/panel/bitrix
```

На ней уже есть автоподстановка текущих значений из сервера, готовый `.env`, готовые URL и короткая инструкция.

## 1. Сгенерируй все данные на странице

1. Открой `https://<SITE_DOMAIN>/panel/bitrix`.
2. В блоке `Шаг 1. Авто Данные Сервера` укажи:
- `SITE_DOMAIN`
- `ACME_EMAIL`
3. Если токены пустые, нажми `Сгенерировать Отсутствующие Токены`:
- токены будут созданы на сервере;
- и сразу сохранены в `.env`.
4. Нажми `Копировать .env Блок`.

## 2. Примени `.env` на сервере

Вставь скопированный блок в `.env`, затем перезапусти:

```bash
docker compose up --build -d
```

Важно:
- DNS `A` запись `SITE_DOMAIN` должна указывать на IP сервера.
- Порты `80` и `443` должны быть открыты.

## 3. Заполни Bitrix без ручного поиска путей

На этой же странице в блоке `Что заполнить в Bitrix локальном приложении` уже готово:
- `Путь вашего обработчика`
- `Путь для первоначальной установки`
- `Webhook Open Lines`
- права приложения: `imconnector` (обязательно), `imopenlines` (для проверки линии в диагностике)

Скопируй и вставь эти данные в Bitrix.

## 4. Подключи портал отдельно

После установки Bitrix откроет эту же страницу с `AUTH_ID/REFRESH_ID/APP_SID` и токены подставятся в поля.

Открой блок `Шаг 3. Подключить Портал (Сайт)` и нажми `Подключить Портал`.

## 5. Привяжи менеджеров к линиям отдельно

Открой блок `Шаг 4. Привязать Менеджера К Линии`.

Для каждого менеджера делай отдельную привязку:
1. Выбери `portal_domain`.
2. Выбери `Менеджер`.
3. Если Telegram-аккаунт уже авторизован, но с другим manager-id:
- нажми `Обновить TG Аккаунты`;
- выбери нужный аккаунт в поле `Telegram аккаунт (gateway)`;
- нажми `Синхронизировать Telegram Аккаунт`.
4. Укажи `line_id`.
5. Нажми `Добавить Привязку Менеджера`.

Этот шаг можно повторять сколько угодно раз для разных менеджеров и линий.

## 6. Проверка

1. Отправь сообщение из Telegram клиенту.
2. Убедись, что оно появилось в Open Lines.
3. Ответь оператором в Open Lines.
4. Убедись, что ответ пришел в Telegram.

Дополнительно в панели (`Шаг 4`) есть кнопки:
- `Проверить Портал API` - делает тестовый REST-запрос в Bitrix и проверяет auth.
- `Проверить Линию` - проверяет привязку менеджера и делает запрос к линии в Bitrix.
- `Синхронизировать Telegram Аккаунт` - исправляет mismatch между manager в gateway и manager в binding.

На странице также есть блок `Логи Интеграции`:
- показывает, что webhook из Telegram получен (`webhook_received`);
- показывает попытку отправки в Bitrix (`send_message_attempt`);
- показывает успешную отправку (`send_message`) или ошибку (`send_message_failed`);
- показывает последние события `telegram-gateway`.

## 7. Важно про HTTPS

- Не используй `https://<домен>:8090`.
- Используй:
  - `https://<SITE_DOMAIN>/panel/bitrix`
  - `https://<SITE_DOMAIN>/telegram/`
