Шаг 3. Подключить Портал (Сайт)

    

      Откуда брать значения: после установки локального приложения Bitrix открывает эту страницу с параметрами `AUTH_ID`, `REFRESH_ID`, `APP_SID`, `DOMAIN`, `MEMBER_ID`, `AUTH_EXPIRES`. Поля ниже заполняются автоматически, если эти параметры есть в URL.
    

    

      

        
        
      

      

        
        
      

      

        
        
      

      

        
        
      

      

        
        
      

      

        
        
      

      

        
        
      

      

        
        
      

      

        
        
      

      

        
        
      

      

        
        
      

    

    

      
    

    

      Подробно: что и откуда брать для каждого поля
      

        
Портал Bitrix (domain)
Что ставить: домен портала без https://
Откуда взять: параметр DOMAIN из URL после установки приложения
Пример: mycompany.bitrix24.ru

        
Access token
Что ставить: AUTH_ID
Откуда взять: параметр AUTH_ID (или access_token) из URL Bitrix
Назначение: текущий OAuth токен для REST вызовов

        
Refresh token
Что ставить: REFRESH_ID
Откуда взять: параметр REFRESH_ID (или refresh_token) из URL Bitrix
Назначение: обновление access token

        
Application token
Что ставить: APP_SID
Откуда взять: параметр APP_SID (или application_token) из URL Bitrix

        
client_endpoint
Что ставить: https://<domain>/rest
Откуда взять: параметр CLIENT_ENDPOINT (если его нет, оставь автозначение)

        
member_id
Что ставить: MEMBER_ID (необязательно)
Откуда взять: параметр MEMBER_ID из URL Bitrix

        
expires_in
Что ставить: AUTH_EXPIRES или 3600
Откуда взять: параметр AUTH_EXPIRES из URL Bitrix

        
scope
Что ставить: imconnector
Откуда взять: оставь значение по умолчанию, если не требуется другое

        
client_id
Что ставить: ID локального приложения (клиент OAuth)
Откуда взять: Bitrix -> Приложения разработчика -> твое локальное приложение -> поле client_id

        
client_secret
Что ставить: секрет локального приложения
Откуда взять: там же, в карточке приложения, поле client_secret

        
server_endpoint
Что ставить: https://oauth.bitrix.info/rest
Откуда взять: можно оставить значение по умолчанию (для облачного Bitrix24)

      

      
Пример URL от Bitrix (сокращенно):
https://<SITE_DOMAIN>/bitrix/app?DOMAIN=mycompany.bitrix24.ru&AUTH_ID=...&REFRESH_ID=...&APP_SID=...&MEMBER_ID=...&AUTH_EXPIRES=...
